# SWGOH Guild Suite — Vision Document

## 1. Vision

### 1.1
SWGOH Guild Suite начинает разрабатываться как персональный инструмент лидера гильдии BuzzLighter.

### 1.2
Смысл разработки — создать удобные для (в первую очередь) себя и своих согильдейцев инструменты управления аккаунтом и гильдией, инструменты ведения статистики, визуализации и обработки любых данных, связанных с игрой.

### 1.3 Основные отличия от других подобных инструментов

1. **Данные в реальном времени** — через open-source Comlink API, а не периодически обновляемые данные от swgoh.gg (которые для большинства данных требуют получения персонального API-ключа).
2. **Личное видение** — смысла и формы инструментов.
3. **Единая модель данных** — доступная из любого инструмента для учёта и обработки разными способами.
4. **Набор инструментов в одном месте с GUI** — а не текстовые команды Discord'а без графической составляющей; многие боты предоставляют только часть функционала, а самые нужные из них — платные.
5. **Полная гарантия невмешательства** — в аккаунты игроков и соблюдение EULA EA.

## 2. Users

1. **Первое и главное** — инструмент создаётся для личного использования.
2. В случае успеха — будет распространён среди офицеров гильдии, а затем и среди остальных игроков.
3. В случае одобрения — доступ будет предоставлен всем игрокам (возможно на коммерческой основе или с монетизацией).
4. **Роли:**
   - **Все игроки** — могут видеть общие данные о гильдии: активность, достижения в событиях и т.п.
   - **Офицеры и лидер** — получают инструменты для планирования гильдейских событий и подробной аналитики складов игроков гильдии.

## 3. Scope

### 3.1 Планировщик снаряжения
- Полная база всех «расходников» игры из Comlink API или публичного репозитория [swgoh-utils/gamedata](https://github.com/swgoh-utils/gamedata).
- Личный склад расходников — только ручной ввод (как на gear.swgohevents.com) или загрузка .json (как на mods-optimizer.swgoh.grandivory.com).
- Максимум информации по каждой детали снаряжения: валюты, фарм, тип энергии и т.д.
- Составление плана с выбором персонажей или готовые наборы согласно «Руководству Путешественника» / отрядам от сообщества.
- Форма предложения отрядов любым игроком с категориями (ВА, ТБ и т.д.).
- Анализ данных: оценка предложений магазина, сравнение шансов выпадения с обычной стоимостью.

### 3.2 Аналитика гильдии
- Активность игроков, ГМ, стаж в гильдии, ежедневные рейдовые купоны, время последней активности.
- Результаты в рейдах, Территориальных битвах, Войне Гильдий.
- Количество определённых юнитов в гильдии.
- Готовность к прохождению «Руководства Путешественника».

### 3.3 Автоматический планировщик ТБ
- Анализ всего ростера гильдии.
- Отсеивание неактивных игроков (1 день или настраиваемый порог).
- Оптимизированное распределение юнитов по взводам/операциям.
- Подсчёт возможных результатов и рекомендации.
- Удобная форма вывода результатов.
- Блок с руководствами (текст, графика, видео).

### 3.4 Планирование Войны Гильдий
- Инструменты для создания и хранения шаблонов отрядов и планов обороны.
- Список-руководство по подбору отрядов для гарантированных/вероятных побед и отрядов, которые не справятся.

### 3.5 Планировщик раунда Великой Арены (последняя очередь)
- Ручное заполнение отрядов противника.
- Анализ ростера и предложение контр согласно правилам (swgoh.gg/gac/counters/) с примерами боёв.

### 3.6 Опции (если проект дорастёт)
- Подключение и обучение LLM для анализа способностей юнитов и стратегий боя.
- Рекомендации по подбору модулей и синергии.

### 3.7 Что НЕ ДЕЛАЕМ
- Никаких инструментов, требующих входа в аккаунт. Только READ ONLY общедоступной информации.

## 4. Supported Data Sources

- **Comlink API** — данные в реальном времени
- **github.com/swgoh-utils/gamedata** — статические данные игры
- **swgoh.gg** — общедоступные данные и графические файлы
- **.json / .csv** — импорт пользовательских файлов (json с пояснениями, csv с макетом)

---

## 5. Core Data Model

| Сущность | Описание |
|---|---|
| **GameData** | Условно статические данные игры (меняются только с обновлениями) |
| **Guild** | Объединение от 1 до 50 игроков |
| **Player** | Данные о конкретном аккаунте |
| **PlayerRoster** | Список юнитов конкретного игрока |
| **GuildRoster** | Сумма всех PlayerRoster, входящих в Guild |
| **Inventory** | Расходные предметы игрока (предоставляются пользователем) |
| **Event** | Событие для игрока или гильдии — временное или периодическое |
| **Bundle** | Внутриигровая поставка/пакет за валюту |
